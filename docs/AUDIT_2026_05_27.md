## §4 Test coverage gaps for v1.0.1 / v1.0.2 work (3 high-leverage tests)

`scripts/probes/*.py` have **zero tests**. `scripts/family_correction.py` has **zero tests** (the underlying `holm_bonferroni` in `src/lerobot_bench/stats.py` is well-tested — `tests/test_stats.py:513–568` — so the analytic risk is in the CLI wrapper, not the math).

Proposed minimal coverage (highest leverage first):

1. **AST guard: no top-level torch/lerobot in `scripts/probes/`.** Mirror `tests/test_run_one.py`'s pattern. Both probe scripts must be importable on a CI box without GPU dependencies. ~30 LoC, prevents the entire pipeline from breaking on a `--help`.
2. **`_patch_act_inference_settings()` unit test (monkey-patch isolation).** Use a stub `PreTrainedConfig` class; assert the patched `from_pretrained` mutates only ACT configs (not DiffusionPolicy / SmolVLA), sets `temporal_ensemble_coeff=0.01` and `n_action_steps=1`, and is idempotent. ~40 LoC.
3. **`dataclasses.replace(env_spec, max_steps=600)` round-trip test** for the smolvla probe — assert the new spec hashes the same fields except `max_steps`, and that the original spec is unchanged (frozen-dataclass discipline). ~20 LoC.

That's it. The probes are one-off, deliberately ephemeral per their own docstrings ("If interesting, promote to PolicySpec.policy_overrides"). 50-test scaffolding would be over-investment.
